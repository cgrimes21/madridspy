add_fingerprint

See also:
Logout proc (mob)
client var (mob)
Format:
add_fingerprint(mob/M)
When:
Called when a player interacts with/touches something, involving picking up objects or opening doors.
Default action:
If the FPRINT flag is not in src.flags, return. If it is, it is able to be finger printed. First we check to see if the fingerprints list is new, if it is we add a hashed value of their real name + "fp"

If there is a fingerprint on the atom already, we turn it into a list, take away your print, and check if the length is over 3. Remove first prints until list equals 3, then add your print back ontop of the list.
